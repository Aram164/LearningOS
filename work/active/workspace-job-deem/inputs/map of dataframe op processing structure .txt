STRATUM DATAFRAME IR: FILES, NODE TYPES, AND FACTORY FUNCTIONS
==============================================================

TERMINOLOGY
===========

IR node:
    An object derived from Op that represents one operation in the graph.

Graph input:
    An upstream Op stored in node.inputs.

Runtime input:
    The actual pandas/polars/NumPy value passed to node.process().

Factory function:
    A make_* function that converts a generic IR node into a specialized node.

OutputType:
    UNKNOWN  = unknown or ordinary Python value
    FRAME    = pandas/polars DataFrame
    SERIES   = pandas/polars Series
    MATRIX   = NumPy ndarray
    SCALAR   = scalar value


HIGH-LEVEL FILE RELATIONSHIPS
=============================

_ops.py
    |
    +-- Defines generic IR nodes:
    |       CallOp, MethodCallOp, GetItemOp, GetAttrOp, BinOp, ...
    |
    v
_dataframe_ops.py
    |
    +-- Inspects generic nodes and dispatches transformations to:
        |
        +-- _source_ops.py
        +-- _projection_ops.py
        +-- _selection_ops.py
        +-- _aggregation_ops.py
        +-- _join_ops.py
        +-- _split_ops.py

_selection_ops.py
    |
    +-- Uses _column_expr.py to fold mask-expression graphs.


======================================================================
1. _ops.py
======================================================================

PURPOSE
-------

Defines the generic IR infrastructure and the generic node types produced
before DataFrame-specific optimization.


SUPPORTING CLASSES
------------------

OutputType(Enum)
    Describes the kind of value produced by an Op.

    Values:
        UNKNOWN
        FRAME
        SERIES
        SCALAR
        MATRIX


OperandRef
    Represents a reference to an entry in an operation's inputs list.

    Example:

        OperandRef(1)

    means:

        use inputs[1]


OperandBinder
    Constructs an operation's inputs list and creates the corresponding
    OperandRef objects.


DummyConfigManager
    No-op context manager used when no estimator parallel configuration
    is needed.


GENERIC IR NODE CLASSES
-----------------------

Op
    Base class for every IR operation.

    Important properties:

        inputs
        outputs
        output_type
        name
        is_X
        is_y

    process(mode, inputs):
        Must be implemented by subclasses.


ImplOp(Op)
    Wraps a generic skrub implementation object.

    Runtime input:
        Values required by the wrapped skrub operation.

    Runtime output:
        Result of skrub_impl.eval() or skrub_impl.compute().


VariableOp(Op)
    Represents a named external variable.

    Runtime behavior:
        Should not reach runtime.

    Expected behavior:
        It must be replaced with a constant ValueOp during compilation.


BaseEstimatorOp(Op)
    Base node for estimator and transformer operations.

    Graph inputs:
        input[0] = X
        optional y
        optional column selector
        optional graph-fed parameters

    Runtime output:
        Estimator or transformer result.


EstimatorOp(BaseEstimatorOp)
    Represents an estimator/predictor.

    Runtime behavior:
        fit_transform mode:
            fit estimator, then predict

        predict mode:
            predict


TransformerOp(BaseEstimatorOp)
    Represents a transformer.

    Runtime behavior:
        fit_transform mode:
            fit and transform

        predict mode:
            transform


ChoiceOp(Op)
    Represents several alternative graph outcomes.

    Graph inputs:
        One input per possible outcome.

    Runtime output:
        One result or a list of named alternative results.

    DataFrame extraction behavior:
        If every outcome is frame-like, propagate their OutputType.
        Mixed FRAME/SERIES outcomes fall back to FRAME.


ValueOp(Op)
    Represents a constant Python value.

    Runtime input:
        None

    Runtime output:
        Stored value

    DataFrame transformation:

        ValueOp containing DataFrame
            |
            +---> DataSourceOp


MethodCallOp(Op)
    Generic method-call node.

    Examples:

        df.drop(...)
        df.groupby(...)
        series.apply(...)
        series.str.lower()

    Graph inputs:

        input[0] = object receiving the method call
        remaining inputs = graph-fed arguments

    Runtime output:

        getattr(inputs[0], method_name)(*args, **kwargs)


CallOp(Op)
    Generic function-call node.

    Examples:

        pd.read_csv(path)
        pd.to_datetime(column)
        np.load(path)

    Graph inputs:
        Values required by args and kwargs.

    Runtime output:

        func(*resolved_args, **resolved_kwargs)


GetAttrOp(Op)
    Generic attribute-access node.

    Examples:

        df.column
        column.str
        column.dt

    Graph input:

        input[0] = object receiving attribute access

    Runtime output:

        getattr(inputs[0], attr_name)


GetItemOp(Op)
    Generic indexing node.

    Examples:

        df["column"]
        df[["a", "b"]]
        df[mask]
        series[index]

    Graph input:

        input[0] = indexed container

    Key:
        Constant key or OperandRef.

    Runtime output:

        inputs[0][key]

    Polars filter special case:

        if is_filter:
            inputs[0].filter(key)


BinOp(Op)
    Generic binary operation.

    Examples:

        column > 10
        column + 1
        mask1 & mask2

    Operands:
        left and right can be constants or OperandRef objects.

    Runtime output:

        op(left, right)


UnaryOp(Op)
    Generic unary operation.

    Examples:

        ~mask
        -column

    Operand:
        Constant or OperandRef.

    Runtime output:

        op(operand)


SearchEvalOp(Op)
    Represents evaluation of grid-search alternatives.

    It is not specifically a DataFrame operation.


MAKE_* FUNCTIONS
----------------

No make_* functions are defined in _ops.py.

The generic nodes in this file are converted by make_* functions located
in the specialized DataFrame IR modules.


======================================================================
2. _source_ops.py
======================================================================

PURPOSE
-------

Defines DataFrame and file-reading source nodes.


NODE CLASSES
------------

DataSourceOp(Op)
    Represents either:

        1. An in-memory DataFrame
        2. A CSV file read
        3. A Parquet file read
        4. A NumPy NPY file read

    Construction forms:

        DataSourceOp(data=df)

        DataSourceOp(
            file_path=...,
            _format="csv" | "parquet" | "npy"
        )

    Graph inputs:
        Optional graph-fed filename and read arguments.

    Runtime output:

        data is supplied:
            pandas DataFrame
            or Polars DataFrame when force_polars=True

        format == "csv":
            pandas/polars DataFrame

        format == "parquet":
            pandas/polars DataFrame

        format == "npy":
            NumPy ndarray

    OutputType:

        csv      -> FRAME
        parquet  -> FRAME
        data     -> FRAME
        npy      -> MATRIX


MAKE_* FUNCTIONS
----------------

make_read_op(op: CallOp, format: str = "csv") -> DataSourceOp

    Input node:
        CallOp

    Expected input patterns:

        CallOp(pd.read_csv)
        CallOp(pd.read_parquet)
        CallOp(np.load)

    Input requirements:
        All op.inputs must be ValueOp or VariableOp.

    Transformation:

        CallOp(pd.read_csv)
            |
            +---> DataSourceOp(format="csv")

        CallOp(pd.read_parquet)
            |
            +---> DataSourceOp(format="parquet")

        CallOp(np.load)
            |
            +---> DataSourceOp(format="npy")

    Constants:
        Inlined directly into DataSourceOp fields.

    Variables:
        Preserved as graph inputs and represented with OperandRef.


OTHER IMPORTANT HELPERS
-----------------------

rechunk_pl_frame(df, rows_per_chunk=128_000)

    Input:
        Polars DataFrame

    Return:
        Polars DataFrame with the same rows and values.

    Physical effect:
        Splits a large frame into bounded physical chunks and concatenates
        them with rechunk=False.

    Logical effect:
        None; data and row order remain unchanged.

3. _projection_ops.py
======================================================================

PURPOSE
-------

Defines operations that compute, select, transform, assign, rename, or
remove columns without primarily representing row filtering.


NODE CLASSES
------------

MetadataOp(Op)
    Represents a metadata-oriented frame method.

    Current use:
        DataFrame.rename(...)

    Graph input:

        input[0] = DataFrame

    Runtime output:
        Renamed DataFrame

    OutputType:
        FRAME


ProjectionOp(Op)
    Base class for projection operations.

    Can represent:

        function-based projection
        or
        method-based projection

    Graph input:

        input[0] = primary DataFrame or Series
        remaining inputs = graph-fed arguments

    Default OutputType:
        FRAME

    Some factories override the type to match input[0].


DropOp(ProjectionOp)
    Represents:

        df.drop(...)

    Graph input:

        input[0] = DataFrame

    Runtime output:
        DataFrame with selected columns or rows removed.

    OutputType:
        FRAME


ColumnSelectorOp(Op)
    Represents deferred skrub column selection.

    Produced from:
        TransformerOp wrapping skrub.SelectCols

    Graph input:

        input[0] = DataFrame

    Runtime output:
        DataFrame containing the selected columns.

    OutputType:
        FRAME

    Special behavior:
        During fit_transform, resolves and stores actual column names.
        During transform, reuses the stored names.


ApplyUDFOp(ProjectionOp)
    Represents:

        frame.apply(function)
        series.apply(function)

    Graph input:

        input[0] = DataFrame or Series

    Runtime output:
        Result of applying the function.

    OutputType during extraction:

        FRAME input  -> FRAME
        SERIES input -> SERIES


AssignOp(ProjectionOp)
    Represents:

        df.assign(...)

    Graph input:

        input[0] = DataFrame

    Runtime output:
        DataFrame with added or replaced columns.

    Pandas execution:
        df.assign(...)

    Polars execution:
        df.with_columns(...)

    OutputType:
        FRAME


DatetimeConversionOp(ProjectionOp)
    Represents:

        pd.to_datetime(value)

    Graph input:

        input[0] = DataFrame or Series being converted

    Runtime output:
        Datetime-converted value.

    Pandas execution:
        pd.to_datetime(...)

    Polars execution:
        inputs[0].str.to_datetime(...)

    OutputType during extraction:
        Same as input[0].


StringMethodOp(ProjectionOp)
    Represents a fused string method.

    Example:

        column.str.lower()

    Before:

        column
           |
           v
        GetAttrProjectionOp(["str"])
           |
           v
        MethodCallOp("lower")

    After:

        column
           |
           v
        StringMethodOp("lower")

    Graph input:

        input[0] = source Series
        remaining inputs = graph-fed method arguments

    Runtime output:
        Result of applying the string method to Series values.

    OutputType:
        Same as the string accessor, normally SERIES.


GetAttrProjectionOp(Op)
    Represents DataFrame/Series attribute access.

    Examples:

        df.column
        column.str
        column.dt
        column.dt.year

    Attribute chain:
        Stored as a list.

    Example:

        ["dt", "year"]

    Graph input:

        input[0] = DataFrame or Series

    Runtime output:
        Result of applying each attribute in sequence.

    OutputType during extraction:
        Same as the original input.


MAKE_* FUNCTIONS
----------------

make_column_selector_op(
    op: TransformerOp
) -> ColumnSelectorOp

    Expected input:
        TransformerOp whose estimator is skrub.SelectCols.

    Transformation:

        TransformerOp(SelectCols)
            |
            +---> ColumnSelectorOp

    OutputType:
        FRAME

    Graph effect:
        Rewires input producers to the new ColumnSelectorOp.


make_datetime_conversion_op(
    op: CallOp
) -> DatetimeConversionOp

    Expected input:
        CallOp representing pd.to_datetime(...).

    Transformation:

        CallOp(pd.to_datetime)
            |
            +---> DatetimeConversionOp

    OutputType:
        Copied from op.inputs[0].

        SERIES -> SERIES
        FRAME  -> FRAME


make_frame_get_attr(
    new_op: GetAttrProjectionOp,
    op: GetAttrOp
) -> GetAttrProjectionOp

    Practical inputs:

        new_op:
            Existing scratch value; can effectively be None when called
            by extract_dataframe_op().

        op:
            GetAttrOp being transformed.

    Case 1: single attribute

        GetAttrOp("column")
            |
            +---> GetAttrProjectionOp(["column"])

    Case 2: chained attributes

        GetAttrProjectionOp(["dt"])
            +
        GetAttrOp("year")
            |
            +---> GetAttrProjectionOp(["dt", "year"])

    OutputType:
        Same as original source input.


make_string_method_op(
    op: MethodCallOp
) -> StringMethodOp

    Expected input shape:

        op.inputs[0] is GetAttrProjectionOp(["str"])

    Transformation:

        GetAttrProjectionOp(["str"])
            +
        MethodCallOp("<string method>")
            |
            +---> StringMethodOp("<string method>")

    OutputType:
        Copied from the string accessor, normally SERIES.

    Graph effect:
        The new StringMethodOp receives the original column directly.
        The ".str" accessor is detached when it has no other consumers.


OTHER IMPORTANT HELPERS
-----------------------

resolve_selector_columns(frame, selector) -> list[str]

    Input:
        DataFrame and skrub selector.

    Return:
        Concrete list of matching column names.

4. _column_expr.py
======================================================================

PURPOSE
-------

Defines an expression tree used inside SelectionOp predicates.

These are NOT Op graph nodes. They are immutable expression values stored
inside specialized nodes.


EXPRESSION CLASSES
------------------

ColumnExpr
    Base class for column expressions.

    Can be evaluated as:

        pandas boolean/value expression
        Polars expression
        pandas query string, when supported


Col(ColumnExpr)
    Represents a column in the source frame.

    Example:

        Col("age")

    Pandas:
        frame["age"]

    Polars:
        pl.col("age")


Const(ColumnExpr)
    Represents a literal constant.

    Examples:

        Const(18)
        Const("Berlin")


OperandLeaf(ColumnExpr)
    Represents an upstream Op that could not be absorbed into the expression.

    Contains:
        OperandRef

    Runtime value:
        inputs[ref.k]


BinOpExpr(ColumnExpr)
    Represents a binary expression.

    Examples:

        Col("age") > Const(18)
        Col("a") + Col("b")
        mask1 & mask2


UnaryOpExpr(ColumnExpr)
    Represents a unary expression.

    Examples:

        ~mask
        -Col("value")


StrExpr(ColumnExpr)
    Represents a string method within a column expression.

    Example:

        StrExpr(Col("name"), "contains", ("abc",))

    Conceptual source:

        df["name"].str.contains("abc")


_Folder
    Internal helper class.

    Purpose:
        Converts an Op subgraph into a ColumnExpr tree.

    It determines:

        foldable nodes
        absorbable nodes
        external leaf operations


MAKE_* FUNCTIONS
----------------

No make_* functions are defined in _column_expr.py.


OTHER IMPORTANT HELPER
----------------------

fold_column_expr(
    root_node: Op,
    src: Op,
    root_consumer: Op
) -> tuple[ColumnExpr, list[Op], list[Op]]

    Inputs:

        root_node:
            Root of the mask/expression subgraph.

        src:
            Source DataFrame operation.

        root_consumer:
            Operation consuming the expression, normally a GetItemOp.

    Return:

        (
            expression,
            absorbed_operations,
            leaf_operations
        )

    Example input graph:

        df["age"] > 18

    Possible expression result:

        BinOpExpr(
            operator.gt,
            Col("age"),
            Const(18)
        )

5. _selection_ops.py
======================================================================

PURPOSE
-------

Defines row-selection and filtering operations.


SUPPORTING ENUM
---------------

SelectionKind(Enum)

    MASK
        Boolean mask filtering.

    QUERY
        Query-style selection kind.
        Currently not created by the listed make_* factories.

    DROPNA
        Remove rows containing null values.

    DROP_DUPLICATES
        Remove duplicate rows.

    HEAD
        Keep first rows.

    TAIL
        Keep last rows.

    SAMPLE
        Select sampled rows.


NODE CLASSES
------------

SelectionOp(Op)
    Represents an operation that restricts rows while keeping the column
    structure.

    Fields:

        kind
        args
        kwargs
        predicate

    Method-based graph input:

        input[0] = DataFrame or Series

    Mask-based graph inputs:

        input[0] = source DataFrame
        remaining inputs = expression leaf operations

    Runtime behavior:

        MASK:
            Evaluate predicate and filter input[0].

        DROPNA:
            pandas: dropna()
            polars: drop_nulls()

        DROP_DUPLICATES:
            pandas: drop_duplicates()
            polars: unique()

        HEAD:
            head()

        TAIL:
            tail()

        SAMPLE:
            sample()

    OutputType:

        Method-based selection:
            Same as input[0].

        Mask selection:
            FRAME.


MAKE_* FUNCTIONS
----------------

make_selection_op(
    op: MethodCallOp
) -> SelectionOp

    Expected methods:

        dropna
        drop_duplicates
        head
        tail
        sample

    Transformations:

        MethodCallOp("dropna")
            |
            +---> SelectionOp(DROPNA)

        MethodCallOp("drop_duplicates")
            |
            +---> SelectionOp(DROP_DUPLICATES)

        MethodCallOp("head")
            |
            +---> SelectionOp(HEAD)

        MethodCallOp("tail")
            |
            +---> SelectionOp(TAIL)

        MethodCallOp("sample")
            |
            +---> SelectionOp(SAMPLE)

    OutputType:
        Copied from op.inputs[0].


make_mask_selection_op(
    op: GetItemOp
) -> SelectionOp

    Expected input shape:

        op represents df[mask]

        op.inputs[0] = source DataFrame
        op.key       = OperandRef pointing to mask operation

    Before:

        DataFrame
           |
           +-------------------+
           |                   |
           v                   v
        GetItemOp("age")    other expression inputs
           |
           v
        BinOp(">")
           |
           v
        GetItemOp(df, mask)

    Transformation:

        fold_column_expr(...)
            |
            v
        SelectionOp(
            kind=MASK,
            predicate=ColumnExpr
        )

    OutputType:
        FRAME

    Graph effect:

        Absorbable mask nodes are detached.
        External leaf nodes remain graph inputs.
        The source frame feeds SelectionOp directly.


OTHER IMPORTANT HELPER
----------------------

is_mask_selection(op: GetItemOp) -> bool

    Returns True when:

        op.key is OperandRef
        and
        op.inputs[0].output_type == FRAME
        and
        mask operation output_type == SERIES

    Important limitation:
        Boolean dtype is not tracked.

    Therefore, any Series-keyed frame indexing may currently be classified
    as mask filtering, even if that Series is not boolean.

6. _aggregation_ops.py
======================================================================

PURPOSE
-------

Defines grouped aggregation operations.


NODE CLASSES
------------

AggregateOp(Op)
    Represents a fused groupby plus aggregation.

    Examples:

        df.groupby("city").mean()
        df.groupby("city").sum()
        df.groupby("city").agg("mean")

    Stored information:

        grouping_attributes
        aggregations
        groupby_kwargs

    Graph inputs:

        input[0] = source DataFrame
        remaining inputs = graph-fed grouping/aggregation arguments

    Runtime behavior:

        input[0].groupby(grouping).agg(aggregations)

    OutputType:
        FRAME

    Polars status:
        Not implemented in AggregateOp.process().


GroupedDataframeOp(Op)
    Experimental grouped DataFrame node.

    Stores:
        List of operations.

    OutputType:
        FRAME

    Runtime status:
        Not integrated.
        process() raises NotImplementedError.


MAKE_* FUNCTIONS
----------------

make_aggregate_op(
    op: MethodCallOp
) -> AggregateOp

    Expected input:
        Aggregation MethodCallOp whose first input is a groupby
        MethodCallOp.

    Accepted patterns:

        groupby(...).sum()
        groupby(...).mean()
        groupby(...).count()
        groupby(...).agg(spec)
        groupby(...).aggregate(spec)

    Before:

        DataFrame
           |
           v
        MethodCallOp("groupby")
           |
           v
        MethodCallOp("mean")

    After:

        DataFrame
           |
           v
        AggregateOp(
            grouping_attributes=...,
            aggregations="mean"
        )

    OutputType:
        FRAME

    Graph effect:
        Bypasses and detaches the intermediate groupby operation.


IMPORTANT RECOGNITION HELPERS
-----------------------------

_is_aggregation(op: MethodCallOp) -> bool

    Returns True only when:

        op directly consumes a groupby MethodCallOp
        groupby has one consumer
        grouping key can be extracted
        method is a supported aggregation


_extract_grouping(groupby_op)

    Input:
        groupby MethodCallOp

    Return:
        Grouping key from positional argument or "by" keyword.


_extract_aggregations(op)

    Input:
        Aggregation MethodCallOp

    Return:

        agg(spec) -> spec
        mean()    -> "mean"
        sum()     -> "sum"
        etc.

7. _join_ops.py
======================================================================

PURPOSE
-------

Defines specialized DataFrame join and merge operations.


NODE CLASSES
------------

JoinOp(Op)
    Represents a binary join between two DataFrames.

    Graph inputs:

        input[0] = left DataFrame
        input[1] = right DataFrame

    Stored configuration:

        how
        left_on
        right_on
        left_index
        right_index
        suffixes

    Runtime output:
        Joined DataFrame

    OutputType:
        FRAME

    Pandas execution:
        left_df.merge(...)

    Polars execution:
        left_df.join(...)

    Important constraint:
        JoinOp expects exactly two runtime inputs.


MAKE_* FUNCTIONS
----------------

make_join_op(
    op: MethodCallOp
) -> JoinOp

    Expected input methods:

        MethodCallOp("join")
        MethodCallOp("merge")

    Input graph:

        op.inputs[0] = left DataFrame
        remaining frame input = right DataFrame(s)

    Transformations:

        MethodCallOp("merge")
            |
            +---> JoinOp

        MethodCallOp("join")
            |
            +---> JoinOp

    Normalizations:

        merge(on="id")
            |
            +-- left_on="id"
            +-- right_on="id"

        join(on="id")
            |
            +-- left_on="id"
            +-- right_index=True

        join without on
            |
            +-- left_index=True
            +-- right_index=True

    OutputType:
        FRAME

    Chained case:

        df1.join([df2, df3])

    Result:
        Returns the final JoinOp in a chain of binary JoinOps.


_make_chained_join_op(
    op: MethodCallOp,
    join_kwargs: dict
) -> JoinOp

    Note:
        Internal factory; its name begins with _make_.

    Input:
        MethodCallOp representing a chained join.

    Before:

        df1.join([df2, df3])

    After:

        df1 ----+
                |
                v
             JoinOp  <---- df2
                |
                v
             JoinOp  <---- df3

    Return:
        Final JoinOp in the chain.

8. _split_ops.py
======================================================================

PURPOSE
-------

Adds an explicit X/y train-test splitting structure to the IR graph.


NODE CLASSES
------------

SplitOp(Op)
    Represents joint selection of X and y using the same row indices.

    Graph inputs:

        input[0] = X
        input[1] = y

    Runtime output:

        (
            X[indices],
            y[indices]
        )

    Supported runtime values:

        pandas DataFrame
        Polars DataFrame
        NumPy ndarray

    Declared OutputType:
        FRAME

    Note:
        The actual runtime result is a pair/tuple.


SplitOutput(Op)
    Extracts one component from SplitOp's tuple.

    Graph input:

        input[0] = SplitOp result

    Configuration:

        is_x=True:
            return inputs[0][0]

        is_x=False:
            return inputs[0][1]

    Declared OutputType:
        FRAME


MAKE_* FUNCTIONS
----------------

No make_* functions are defined in _split_ops.py.


OTHER IMPORTANT GRAPH FUNCTION
------------------------------

add_splitting_op(root: Op) -> Op

    Input:
        Root operation of an IR graph.

    Searches for:

        op.is_X == True
        op.is_y == True

    Inserts:

                  +----------------+
        X ------->|                |------> SplitOutput(X)
                  |    SplitOp     |
        y ------->|                |------> SplitOutput(y)
                  +----------------+

    Return:
        The same graph root, with the splitting nodes inserted.

9. _dataframe_ops.py
======================================================================

PURPOSE
-------

Acts as the central dispatcher for DataFrame-specific extraction.

It imports and re-exports specialized nodes from:

    _source_ops.py
    _projection_ops.py
    _selection_ops.py
    _aggregation_ops.py
    _join_ops.py
    _split_ops.py


NODE CLASSES DEFINED DIRECTLY IN THIS FILE
------------------------------------------

ConcatOp(Op)
    Represents DataFrame concatenation.

    Stored fields:

        first
        others
        axis

    Graph inputs:
        DataFrames or Series referenced through OperandRef objects.

    Runtime output:
        Concatenated DataFrame.

    Pandas execution:

        pd.concat([first, *others], axis=axis)

    Polars execution:

        axis == 0:
            pl.concat(..., how="diagonal_relaxed")

        axis == 1:
            pl.concat(..., how="horizontal")

    OutputType:
        FRAME


MAKE_* FUNCTIONS DEFINED DIRECTLY IN THIS FILE
----------------------------------------------

No make_* functions are defined directly in _dataframe_ops.py.

Instead, extract_dataframe_op() calls imported factory functions.


DISPATCHER FUNCTION
-------------------

extract_dataframe_op(
    op: Op,
    root: Op,
    selection_op=True
) -> tuple[Op, bool]

    Input:

        op:
            Generic or partially specialized IR node being inspected.

        root:
            Current graph root.

        selection_op:
            Whether df[mask] should become SelectionOp(MASK).

    Return:

        (
            possibly updated root,
            whether a replacement node was created
        )


FACTORY DISPATCH TABLE
----------------------

Generic input node/pattern             Factory                       Result
-------------------------------------------------------------------------------

        +---> DropOp(...)

        +---> ApplyUDFOp(...)

    MethodCallOp("assign")

        +---> AssignOp(...)


TYPE-ONLY BRANCHES
------------------

Some branches do not replace the node.

groupby:

    MethodCallOp("groupby")
        |
        +-- remains MethodCallOp
        +-- output_type = FRAME


BinOp / UnaryOp:

    output_type = op.inputs[0].output_type


Ordinary GetItemOp:

    SERIES[key]
        -> SERIES

    FRAME["column"]
        -> SERIES

    FRAME[["a", "b"]]
        -> FRAME

    FRAME[mask or slice]
        -> FRAME


BaseEstimatorOp:

    output_type = FRAME


ChoiceOp:

    all SERIES -> SERIES
    all FRAME  -> FRAME
    mixed      -> FRAME


FINAL REPLACEMENT LOGIC
-----------------------

If no new node was created:

    new_op is None
        |
        +-- return root, False


If a new node was created:

    old op
       |
       +-- downstream consumers

    becomes:

    new op
       |
       +-- downstream consumers

Using:

    op.replace_input_of_outputs(new_op)

If the old operation was the root:

    root = new_op

Return:

    return root, True
