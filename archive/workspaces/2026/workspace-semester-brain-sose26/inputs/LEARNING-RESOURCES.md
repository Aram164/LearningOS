## §0 Intake Protocol (formalized 2026-07-10 — how EVERY new find enters)

Every entry, in any section, carries **four fields**: **type** (📒 book/text · 🎥 video/course · 🧪 solved practice · 📄 paper · 🔧 tool/repo) · **status** · **tags** (table above) · **one-line why**.

**Status vocabulary (exactly one per entry):**

| Status | Meaning | Machine check |
|---|---|---|
| ✓ **local**: `root/relative/path.pdf` | file lives in the owning module's material folder | path validated by `Masters-Planning/tools/check_system.py` |
| 🌐 link | web-native (courses, blogs, interactive, ALL videos) | URL checked by `lychee .` |
| ⬇️ pull | free & downloadable but not yet local — **the download queue** | `grep ⬇️` = open queue |
| 🏛️ license | free via HU/TU/Springer campus login — Aram pulls manually | — |
| 💰 paid | buy decision pending (rule 4) | — |

**Intake steps (30 seconds):** ① add the one-line entry in the right section with type + status + tags → ② if it's a free file: download into the **owning module's material folder, into its type subfolder** (layout below), flip status to ✓ local with the path → ③ done.

**Canonical material-folder layout (enforced 2026-07-10, every module):** `Lecture-slides/` (course decks) · `Buecher/`/`Books/` (books & long texts) · `Klausuren-extern/` (solved exams from other unis) · `papers/` · exercises keep their course folder (`CS4780-homeworks/`, `Bonus-exercises/`, `Drill-Loesungen/` …) · `notes/` (tier-3 units). **Videos are never files** — always 🌐 rows here (sole exception: pre-existing `SaD-2025/` recordings). A file loose at a module folder's top level = mess → move it into its type subfolder. Crosswalks and Mini Plans *select* from here — they never re-list. The promotion ritual sweeps leftover ⬇️/🏛️ and promotes degree-relevant finds to `Masters-Planning/MASTERS-*-RESOURCES.md` axis files.

**Download-vs-link rule of thumb:** exam-critical or thesis-relevant *files* get pulled local (survives dead links, works offline, validator-indexed); casual/broad material stays 🌐.

| [Hettinger — *Python's super() considered super!*](https://rhettinger.wordpress.com/2011/05/26/super-considered-super/) ⭐ (~45 min) | THE inheritance/MRO mental model. Has a [PyCon talk version](https://nedbatchelder.com/text/names1.html) if video preferred. | `PY` `M` |
